## 🏗️ 系统架构概览

### 技术栈
- **前端**：Next.js 15.5.2 + React 19 + TypeScript 5 + Tailwind CSS 4
- **后端**：Node.js + Fastify 4 + Prisma 5 + TypeScript
- **数据库**：PostgreSQL 14 + Redis 7
- **容器化**：Docker + Docker Compose
- **开发工具**：Turbopack、ESLint 9、Prettier

### 服务架构
```
┌─────────────────┐    ┌─────────────────┐    ┌─────────────────┐
│   Frontend      │    │   Backend API   │    │   Database      │
│   Next.js       │◄──►│   Fastify       │◄──►│   PostgreSQL    │
│   Port: 3000    │    │   Port: 4000    │    │   Port: 5432    │
```

## 🔧 环境类型说明

### 开发环境 (Development)
- **用途**：本地开发、功能调试、快速迭代
- **特点**：热重载、详细日志、开发工具支持
- **数据库**：本地Docker容器或云开发数据库
- **安全级别**：低（使用测试密钥）

### 测试环境 (Testing/Staging)
- **用途**：功能测试、集成测试、预上线验证
- **特点**：模拟生产环境、测试数据隔离
- **数据库**：独立测试数据库
- **安全级别**：中（使用测试密钥，但接近生产配置）

### 生产环境 (Production)
- **用途**：正式运营、用户访问
- **特点**：高性能、高可用、完整监控
- **数据库**：生产级数据库集群
- **安全级别**：高（强密钥、加密传输、访问控制）

## 🔐 密钥管理方式

### 密钥分类与安全级别

#### 🔴 高安全级别密钥
- **JWT_SECRET**: JWT token 签名密钥
- **DATABASE_URL**: 数据库连接字符串
- **生产环境第三方API密钥**: 阿里云、微信等生产密钥
- **NEXTAUTH_SECRET**: NextAuth.js 认证密钥

**管理方式**：
- 使用专业密钥管理服务（AWS Secrets Manager、Azure Key Vault、阿里云KMS）
- 定期轮换（每90天）
- 访问日志审计
- 最小权限原则

#### 🟡 中安全级别密钥
- **REDIS_PASSWORD**: Redis 访问密码
- **测试环境API密钥**: 测试用第三方服务密钥
- **对象存储密钥**: 文件存储访问密钥

**管理方式**：
- 环境变量注入
- Git 仓库加密存储
- 定期更新（每180天）

#### 🟢 低安全级别密钥
- **开发环境数据库密码**: 本地开发用密码
- **开发环境服务密钥**: 开发调试用密钥

**管理方式**：
- 本地环境变量文件
- 团队共享文档
- 相对较少更新

### 密钥管理最佳实践

#### 1. 密钥生成规范
```bash
# JWT Secret 生成 (64字符随机字符串)
node -e "console.log(require('crypto').randomBytes(64).toString('hex'))"

# NextAuth Secret 生成 (32字符随机字符串)
openssl rand -base64 32

# 数据库密码生成 (16字符复杂密码)
openssl rand -base64 16 | tr -d "=+/" | cut -c1-16
```

#### 2. 环境隔离策略
```bash
# 开发环境密钥前缀
DEV_*

# 测试环境密钥前缀
STAGING_*

# 生产环境密钥前缀
PROD_*
```

#### 3. 密钥轮换策略
- **生产环境**: 每30-90天轮换关键密钥
- **测试环境**: 每6个月轮换
- **开发环境**: 每年轮换或安全事件后轮换

#### 4. 访问控制矩阵
```
密钥类型          开发人员  运维人员  安全管理员  系统管理员
开发环境密钥        ✅       ✅       ✅        ✅
测试环境密钥        ❌       ✅       ✅        ✅
生产环境密钥        ❌       ❌       ✅        ✅
密钥轮换权限        ❌       ❌       ✅        ✅
```

## 🚀 初始化和启动步骤

### 开发环境初始化

#### 1. 环境准备
```bash
# 1.1 安装系统依赖
# macOS
brew install node@20 postgresql@14 redis@7 docker

# Ubuntu/Debian  
sudo apt update
sudo apt install nodejs npm postgresql-14 redis-server docker.io docker-compose

# Windows (使用 WSL2)
# 安装 Docker Desktop for Windows
# 使用 WSL2 Ubuntu 子系统
```

#### 1.2 项目克隆和依赖安装
```bash
# 克隆项目
git clone https://github.com/your-org/headhunter-platform.git
cd headhunter-platform

# 安装根依赖
npm install

# 安装前端依赖
cd frontend
npm install

# 安装后端依赖
cd ../backend
npm install
```

#### 1.3 环境配置文件设置
```bash
# 复制环境变量模板
cp backend/.env.example backend/.env
cp frontend/.env.local.example frontend/.env.local

# 编辑配置文件
vim backend/.env          # 配置数据库和服务参数
vim frontend/.env.local   # 配置前端参数
```

#### 1.4 数据库初始化
```bash
# 方式1: 使用 Docker (推荐)
docker-compose up -d postgres redis

# 等待数据库启动
sleep 10

# 运行数据库迁移
cd backend
npm run db:generate      # 生成 Prisma 客户端
npm run db:migrate:dev   # 执行数据库迁移
npm run db:seed          # 初始化种子数据

# 方式2: 本地数据库
# 启动本地 PostgreSQL 和 Redis
brew services start postgresql@14
brew services start redis

# 创建数据库
createdb headhunter_platform_dev

# 执行迁移
npm run db:migrate:dev
npm run db:seed
```

#### 1.5 启动开发服务
```bash
# 方式1: 使用 Docker Compose (推荐)
docker-compose up

# 方式2: 本地启动
# 启动后端 (终端1)
cd backend
npm run dev

# 启动前端 (终端2)  
cd frontend
npm run dev

# 访问地址
# 前端: http://localhost:3000
# 后端API: http://localhost:4000
# API文档: http://localhost:4000/docs
# pgAdmin: http://localhost:5050
# Redis Commander: http://localhost:8081
```

### 测试环境部署

#### 2.1 服务器准备
```bash

# CPU: 2核心以上
```

```bash
# 从 Git 拉取代码
git clone https://github.com/your-org/headhunter-platform.git
cd headhunter-platform

git checkout staging

# 配置环境变量
cp docker-compose.staging.yml docker-compose.yml
```

#### 2.3 数据库和服务启动
```bash

docker-compose build

docker-compose exec backend npm run db:migrate
docker-compose exec backend npm run db:seed

# 检查服务状态
docker-compose ps
docker-compose logs
```

#### 2.4 域名和SSL配置
```bash
# 使用 Nginx 反向代理
sudo apt install nginx certbot python3-certbot-nginx

# 配置 Nginx
sudo vim /etc/nginx/sites-available/headhunter-staging

# Nginx 配置示例
server {
    listen 80;
    server_name staging.headhunter-platform.com;
    
    location / {
        proxy_pass http://localhost:3000;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }
    
    location /api {
        proxy_pass http://localhost:4000;
        proxy_http_version 1.1;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }
}

# 启用站点
sudo ln -s /etc/nginx/sites-available/headhunter-staging /etc/nginx/sites-enabled/
sudo nginx -t
sudo systemctl reload nginx

# 申请 SSL 证书
sudo certbot --nginx -d staging.headhunter-platform.com
```

```bash
# 检查后端服务状态
curl http://localhost:4000/health

# 检查环境变量
echo $NEXT_PUBLIC_API_URL

# 检查网络连接
netstat -tulnp | grep :4000
```

#### 2. 数据库连接失败
```bash
# 检查数据库连接
psql "postgresql://username:password@localhost:5432/dbname"

# 检查 Docker 容器状态
docker-compose ps
docker-compose logs postgres

# 检查数据库迁移状态
npm run db:migrate:status
```

#### 3. Redis 缓存问题
```bash
# 检查 Redis 连接
redis-cli ping

# 检查 Redis 容器
docker-compose logs redis

# 清除缓存
redis-cli FLUSHALL
```

#### 4. 文件上传失败
```bash
# 检查上传目录权限
ls -la uploads/
chmod 755 uploads/

# 检查磁盘空间
df -h

# 检查文件大小限制
grep MAX_FILE_SIZE .env
```

## 🛡️ 安全配置要求

### 网络安全
- 使用 HTTPS/TLS 1.3 加密传输
- 配置防火墙规则，只开放必要端口
- 启用 CORS 策略，限制跨域访问
- 配置 CSP (Content Security Policy)
- 启用 HSTS (HTTP Strict Transport Security)

### 数据安全  
- 数据库连接使用 SSL 加密
- 敏感数据字段级加密存储
- 定期数据备份和恢复测试
- 数据访问审计日志
- 符合 GDPR/CCPA 数据保护要求

### 应用安全
- JWT token 定期轮换
- API 请求频率限制
- SQL 注入防护
- XSS 攻击防护  
- CSRF 攻击防护
- 文件上传安全扫描
