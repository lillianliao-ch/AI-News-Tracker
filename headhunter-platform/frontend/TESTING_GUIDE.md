# 职位分享功能测试指南

## 功能概述

本指南包含以下功能的测试：
1. **职位分享功能** - 通过微信分享职位给候选人
2. **微信联系功能** - 在联系候选人模块中添加微信选项

## 环境要求

- 前端服务：`http://localhost:3003`
- 后端服务：`http://localhost:4000` (或 4002)
- 数据库：包含测试职位和候选人数据

## 测试步骤

### 1. 职位分享功能测试

#### 1.1 准备测试数据
```bash
# 确保后端服务运行
cd /Users/lillianliao/notion_rag/headhunter-platform/backend
npm run dev

# 确保前端服务运行
cd /Users/lillianliao/notion_rag/headhunter-platform/frontend  
npm run dev
```

#### 1.2 测试职位分享按钮
1. 访问 `http://localhost:3003`
2. 登录系统
3. 进入"职位管理"页面
4. 查看职位列表中的"操作"列
5. **预期结果**：对于状态为"approved"或"open"的职位，应该显示"🔗 分享"按钮

#### 1.3 测试分享链接生成
1. 点击任意一个"🔗 分享"按钮
2. **预期结果**：
   - 弹出提示"分享链接已复制到剪贴板!"
   - 剪贴板中包含格式为 `http://localhost:3003/jobs/share/{job-id}` 的链接

#### 1.4 测试公开分享页面
1. 复制获得的分享链接
2. 在新的浏览器窗口/隐身模式中打开链接
3. **预期结果**：
   - 页面无需登录即可访问
   - 显示职位详细信息（职位概况、描述、要求、公司信息）
   - 移动端友好的响应式设计
   - 显示"联系猎头顾问"按钮

#### 1.5 测试邮件联系功能
1. 在分享页面点击"联系猎头顾问"按钮
2. **预期结果**：
   - 打开默认邮件客户端
   - 邮件地址为发布职位的猎头邮箱
   - 主题预填为"关于{职位名称}职位的咨询"
   - 正文包含职位相关信息

#### 1.6 测试微信分享优化
1. 在微信中发送分享链接
2. **预期结果**：
   - 链接预览显示职位标题
   - 显示职位描述、薪资、地点等关键信息
   - 显示"猎头协作平台"品牌信息

### 2. 微信联系功能测试

#### 2.1 进入候选人管理
1. 登录系统
2. 进入"候选人管理"页面
3. 点击任意候选人的"联系"按钮

#### 2.2 测试微信联系选项
1. 在联系候选人弹窗中，查看联系方式选项
2. **预期结果**：
   - 显示三个选项：📞电话联系、📧邮件联系、💬微信联系
   - 微信选项有绿色边框悬停效果

#### 2.3 测试微信联系表单
1. 点击"💬微信联系"选项
2. **预期结果**：显示微信联系表单，包含以下字段：
   - 微信号/昵称
   - 沟通时长
   - 沟通结果（下拉选择）
   - 沟通内容（必填）
   - 下次跟进时间
   - 备注
   - 微信沟通提示信息

#### 2.4 测试表单验证
1. 不填写"沟通内容"，点击"完成沟通记录"
2. **预期结果**：显示错误提示"请填写沟通内容"

#### 2.5 测试微信沟通记录保存
1. 填写完整的微信沟通表单：
   ```
   微信号/昵称: 张三_WeChat
   沟通时长: 20分钟
   沟通结果: 感兴趣
   沟通内容: 候选人对职位很感兴趣，询问了薪资待遇和工作地点的详细信息。表示愿意进一步了解。
   下次跟进时间: [设置一个未来时间]
   备注: 候选人比较关心远程工作的可能性
   ```
2. 点击"完成沟通记录"
3. **预期结果**：
   - 显示"保存中..."状态
   - 成功后关闭弹窗
   - 刷新候选人的沟通记录列表

#### 2.6 验证沟通记录显示
1. 重新打开该候选人的详情页面
2. 查看"沟通记录"部分
3. **预期结果**：
   - 显示新添加的微信沟通记录
   - 记录类型显示为"💬 微信沟通"
   - 包含紫色标签
   - 显示所有填写的信息

## 可能的问题及解决方案

### 1. 分享链接无法访问
- 检查后端API `/api/jobs/share/:id` 是否正常
- 确认数据库中有对应的职位数据

### 2. 剪贴板复制失败
- 在HTTPS环境下测试
- 检查浏览器剪贴板权限

### 3. 微信沟通记录保存失败
- 检查后端API `/api/candidates/{id}/communications`
- 确认communicationType支持'wechat'类型

### 4. 分享页面样式问题
- 检查Tailwind CSS是否正确加载
- 确认响应式断点设置

## 性能测试

### 分享页面加载速度
- 首次加载应在2秒内完成
- 无需登录验证减少了加载时间

### 沟通记录保存速度
- 保存操作应在1秒内完成
- 大量文本内容不应影响性能

### 分享页面安全性
- 确认只显示公开职位信息
- 不泄露内部敏感数据

- 输入字段应有适当的长度限制
- 防止XSS攻击的内容过滤
