## 一、项目概述

### 1.1 项目定位 🎯

**核心使命**：搭建猎头协作的统一工作台，解决"多公司协同难、协同效率"的痛点，实现职位/候选人/结算的闭环协同。

**服务人群**：
- 平台管理员
- Rupro/PM（项目经理）
- 合作公司管理员及顾问
- SOHO顾问

### 1.2 平台特色 ✅

- **统一工作台**：集成所有协作功能，一站式解决方案
- **闭环协同**：职位发布→候选人匹配→面试跟进→入职确认→结算分配的完整闭环
- **多公司协作**：支持跨公司、跨团队的无缝协作
- **智能化驱动**：AI智能匹配、文档识别、自动化流程
- **精细权限管控**：基于角色和项目的精确权限分配

### 1.3 核心价值主张 💎

#### 对猎头公司的价值
- **效率提升 3-5倍**：统一工作台减少系统切换，智能匹配提升精准度
- **协作范围扩大**：从单打独斗到多公司协同，资源最大化利用
- **管理可视化**：项目进度、收益分配、团队绩效一目了然
- **风险可控**：精细权限控制，客户隐私有保障

#### 对顾问个人的价值
- **职位资源翻倍**：接入多公司职位池，机会更多
- **收益透明化**：分成规则清晰，结算自动化
- **专业工具支持**：AI简历解析、智能推荐、沟通记录管理
- **个人品牌积累**：成功案例记录，专业能力可量化

## 二、用户角色体系

### 2.1 平台管理员 (Platform Admin) ✅ **已完成**

- **核心职责**：平台整体运营管理，生态健康维护
- **管理权限**：
  - 审核猎头公司管理员和SOHO顾问注册申请
  - 管理平台整体数据（职位、简历、协作记录）
  - 处理违规操作（拉黑、封禁、争议仲裁）
  - 维护系统配置（通知策略、权限规则、分成比例）
  - 审核候选人维护人变更申请 ❌ **未实现**
  - 监控平台运营数据和业务指标

### 2.2 Rupro/PM (项目经理) ❌ **待开发**

- **角色定位**：大型招聘项目的专项管理者和协调者
- **核心职责**：
  - **项目统筹管理**：创建、配置和管理专项招聘项目
  - **跨公司协调**：协调多个合作公司参与项目执行
  - **进度跟踪监控**：实时跟踪项目进度，确保按时交付
  - **资源优化配置**：合理分配顾问资源，最大化项目效率
  - **质量把控**：审核候选人质量，确保项目交付标准
- **权限范围**：
  - 项目创建和配置权限
  - 跨公司顾问协调权限
  - 项目数据查看和分析权限
  - 项目相关决策和审批权限
- **管理对象**：招聘项目、项目团队、项目资源、项目绩效

### 2.3 公司管理员 (Company Admin) ✅ **已完成**

- **注册流程** ✅：输入企业信息、营业执照、联系人信息，平台管理员审核
- **核心职责**：公司内部管理和业务统筹
- **管理功能** ✅：
  - 创建和管理咨询顾问账号
  - 审核咨询顾问加入申请
  - 配置PM权限（为顾问分配客户访问权限）⭐ **新增功能**
  - 查看本公司相关的职位、简历、协作数据
  - 管理客户公司合作关系
  - 监控团队绩效和收益分配

### 2.4 咨询顾问 (Consultant) ✅ **已完成**

- **注册方式** ✅：申请加入特定猎头公司，由公司管理员审核
- **核心职责**：执行具体的招聘业务
- **业务权限** ✅：
  - 根据PM权限访问指定客户的职位
  - 管理候选人简历和投递记录
  - 参与职位协作和候选人推荐
  - 跟踪候选人状态和沟通记录
  - 查看收益分配和结算明细

### 2.5 SOHO顾问 ✅ **已完成**

- **独立身份**：无公司隶属的独立顾问
- **业务权限**：与咨询顾问基本相同，但无公司内部协作功能
- **特殊优势**：可以与多个公司开展协作，灵活性更高

## 四、业务流程设计

### 4.1 典型业务场景

#### 场景1：统一工作台协作流程
```
平台管理员创建协作项目 
→ Rupro/PM接管项目管理
→ 分配给多个合作公司参与
→ 各公司顾问在统一工作台协同工作
→ 实时进度跟踪和协调
→ 实现职位/候选人/结算的闭环协同
```

#### 场景2：职位大厅高效匹配流程
```
职位发布到平台
→ 自动进入职位大厅展示
→ AI智能推荐给合适顾问
→ 顾问快速响应和认领
→ 候选人精准匹配投递
→ 提升整体协同效率
```

#### 场景3：项目化管理流程
```
大型招聘需求识别
→ 创建专项招聘项目
→ Rupro/PM统筹项目管理
→ 多公司协同团队参与
→ 进度透明化实时跟踪
→ 结算自动化批量处理
```

#### 场景4：PM权限精细管控流程
```
公司管理员识别权限需求
→ 为顾问分配特定客户访问权限
→ 顾问只能看到被分配客户的职位
→ 确保客户隐私和商业机密安全
→ 支持权限的动态调整
```

### 4.2 核心业务流程

#### 4.2.1 用户注册到业务开展流程
```
用户注册申请 → 身份审核 → 权限分配 → 平台培训 → 业务开展
```

#### 4.2.2 职位从发布到成交流程
```
职位发布 → 审核通过 → 职位大厅展示 → 顾问认领 → 候选人投递 → 面试跟进 → 成功入职 → 结算分成
```

```
简历上传 → AI智能解析 → 候选人建档 → 标签化处理 → 智能匹配 → 投递推荐 → 跟进服务 → 成功入职
```

## 十、风险评估与应对

### 10.1 技术风险 ⚠️

#### 高风险项
- **系统性能瓶颈**：用户增长可能导致的性能问题
  - *应对策略*：架构优化、分库分表、缓存策略
- **数据安全风险**：客户敏感信息的安全保护
  - *应对策略*：加密传输、访问审计、权限细分

#### 中风险项  
- **第三方依赖**：AI服务、支付接口等外部依赖的稳定性
  - *应对策略*：多供应商备选、服务降级机制
- **技术债务累积**：快速开发过程中的代码质量问题
  - *应对策略*：定期重构、代码审查、技术规范

### 10.2 业务风险 ⚠️

#### 高风险项
- **市场接受度**：新模式的市场教育和用户接受度
  - *应对策略*：分阶段推广、标杆客户培养、持续优化
- **竞争对手**：大厂或传统软件公司的竞争
  - *应对策略*：技术领先、服务差异化、用户粘性建设

#### 中风险项
- **法律合规**：数据保护、劳动法等法规变化
  - *应对策略*：法务顾问、合规审查、及时调整
- **商业模式**：收费模式和盈利能力的验证
  - *应对策略*：多种模式测试、用户价值验证

### 10.3 运营风险 ⚠️

- **用户流失**：产品体验或服务问题导致的用户流失
  - *应对策略*：用户反馈机制、快速响应、持续改进
- **数据质量**：用户输入数据的准确性和完整性
  - *应对策略*：数据校验、质量评分、激励机制
- **客服压力**：用户量增长带来的客服和支持压力
  - *应对策略*：知识库建设、自助服务、AI客服
