# 同义词配置维护指南

## 📋 概述

同义词配置用于解决 AI 职位匹配中术语不一致的问题，例如：
- `三维重建` 与 `3D重建` 是同一概念
- `TTS` 与 `语音合成` 是同一概念
- `Agent` 与 `智能体` 是同一概念

通过同义词映射，系统能够识别这些等价术语，提高匹配精度。

## 🔧 日常维护操作

### 1. 添加新的同义词组

打开 `config/synonyms.yaml`，在对应分类下添加：

```yaml
# 在 specialties 下添加新的专业方向
- canonical: "自动驾驶"     # 标准形式
  aliases:                  # 别名列表
    - "autonomous driving"
    - "自动驾驶系统"
    - "无人驾驶"
    - "l4"
```

### 2. 扩展现有同义词组

找到对应的同义词组，在 `aliases` 列表中添加新别名：

```yaml
# 原配置
- canonical: "语音合成"
  aliases:
    - "tts"
    - "text-to-speech"
    
# 扩展后
- canonical: "语音合成"
  aliases:
    - "tts"
    - "text-to-speech"
    - "speech synthesis"    # 新增
    - "语音生成"             # 新增
```

### 3. 验证配置

修改后运行验证工具：

```bash

python3 scripts/validate_synonyms.py
```

## ⚠️ 注意事项

### 1. 避免重复词汇

**错误示例：** 同一个词出现在多个分类中

```yaml
# ❌ 错误：agent 同时出现在两个地方
specialties:
  - canonical: "Agent/智能体开发"
    aliases: ["agent", "智能体"]

tech_domains:
  - canonical: "Agent/智能体"
    aliases: ["agent", "智能体"]  # 重复！
```

**正确做法：** 使用不同的别名区分

```yaml
# ✅ 正确：使用不同别名区分上下文
specialties:
  - canonical: "Agent/智能体开发"
    aliases: ["agent", "智能体", "ai agent"]

tech_domains:
  - canonical: "Agent/智能体"
    aliases: ["agent方向", "智能体方向"]  # 区分上下文
```

### 2. 标准形式命名规范

- 使用中文为主，英文缩写放在 aliases
- 格式：`主术语/备用术语`，如 `RAG/知识库`
- 保持一致性，便于阅读

### 3. 配置生效时机

- 修改 `synonyms.yaml` 后**无需重启服务**
- 下次匹配请求时自动加载新配置
- 支持热更新
