# 脉脉招聘助手 - 产品需求文档 (PRD)

## 📋 项目概述

### 项目背景
猎头行业在寻找合适候选人时，需要在脉脉等平台上手动浏览大量人才信息，逐个点击查看详情并记录关键信息。这个过程耗时且容易出错，迫切需要自动化工具来提升效率。

### 产品定位
**脉脉招聘助手**是一款专为猎头设计的Chrome浏览器扩展插件，能够自动批量提取脉脉招聘板块的人才信息，一键操作完成数据收集和导出。

### 核心价值
- **效率提升**：将手动1小时的工作缩短到5分钟自动完成
- **数据准确**：95%的信息提取准确率，避免人工录入错误
- **规模化处理**：单次处理500+人才，满足大型招聘需求

## 🎯 用户分析

### 目标用户
- **主要用户**：猎头/招聘顾问
- **次要用户**：企业内部HR、招聘经理

### 用户画像
```
张小猎 - 资深猎头顾问
- 年龄：28-35岁
- 经验：3-8年招聘行业经验
- 工作强度：每天需要筛选100-500份人才资料
- 技术水平：基础计算机操作，偏好简单易用工具
- 痛点：手动复制粘贴效率低，容易遗漏关键信息
```

### 使用场景
1. **日常人才搜索**：根据职位要求搜索匹配的候选人
2. **人才库建设**：批量收集特定领域的人才信息建立数据库
3. **竞争对手调研**：了解目标公司的人才构成和流动情况
4. **薪资市场调研**：收集行业薪资和人才分布数据

## 🚀 功能需求

### 核心功能

#### 1. 人才信息批量提取 (P0)
**功能描述**：自动识别页面上的人才卡片，逐个点击并提取详细信息

**具体要求**：
- 自动识别人才列表中的所有人才卡片
- 模拟点击打开人才详情弹窗
- 提取完整的人才信息（详见数据字段要求）
- 支持连续处理，无需人工干预

**验收标准**：
- 能识别页面上所有可见的人才卡片
- 弹窗检测准确率 > 98%
- 单个人才处理时间 < 5秒
- 连续处理不出现卡死现象

#### 2. 智能页面导航 (P0)
**功能描述**：自动处理页面滚动和翻页，确保能遍历所有人才

**具体要求**：
- 自动滚动到页面底部加载更多人才
- 检测并点击"跳转至下一页"按钮
- 处理页面加载等待和超时
- 支持暂停和恢复功能

**验收标准**：
- 能自动加载所有可用的人才数据
- 翻页成功率 > 95%
- 页面加载超时自动重试
- 支持用户随时暂停操作

#### 3. 数据质量保证 (P0)
**功能描述**：确保提取数据的准确性和完整性

**具体要求**：
- 多重验证机制确保数据准确性
- 智能重试机制处理提取失败
- 实时数据质量监控和报告
- 支持手动数据校正

**验收标准**：
- 整体数据准确率 ≥ 95%
- 必填字段完整率 ≥ 90%
- 提供详细的质量报告
- 错误数据支持重新提取

#### 4. Excel数据导出 (P0)
**功能描述**：将提取的人才数据导出为标准Excel格式

**具体要求**：
- 标准化的Excel格式输出
- 支持自定义字段选择
- 文件命名包含时间戳
- 支持中文字符正确显示

**验收标准**：
- Excel文件可在主流办公软件中正常打开
- 中文字符显示正常
- 数据格式规整，便于后续处理
- 文件大小合理（500人才 < 5MB）

### 辅助功能

#### 5. 实时进度监控 (P1)
**功能描述**：提供详细的处理进度和统计信息

**具体要求**：
- 实时显示已处理/总数量
- 显示当前处理速度和预估剩余时间
- 提供成功率和错误统计
- 支持处理历史记录

#### 6. 灵活配置选项 (P1)
**功能描述**：允许用户根据需求调整工具行为

**具体要求**：
- 处理间隔时间设置（防反爬）
- 最大处理数量限制
- 数据字段选择开关
- 错误处理策略选择

#### 7. 异常恢复机制 (P1)
**功能描述**：在异常情况下保护数据和恢复处理

**具体要求**：
- 自动保存处理进度
- 支持断点续传
- 异常情况下的数据保护
- 错误日志记录和分析

## 📊 数据需求

### 人才信息字段要求

#### 基础信息 (必提取字段)
```
- 姓名：人才真实姓名或昵称
- 工作地点：当前工作城市
- 工作年限：总工作经验，如"3年"、"5-10年"
- 年龄：具体年龄数字
- 学历：最高教育水平，如"本科"、"硕士"
```

#### 详细信息 (尽力提取字段)
```
工作经历：
- 公司名称：历任公司的完整名称
- 职位名称：具体职位title
- 工作时间：起止时间，如"2020.01-2023.05"
- 工作描述：职责和成就描述

教育经历：
- 学校名称：毕业院校完整名称
- 专业名称：所学专业
- 学历层次：学士/硕士/博士
- 就读时间：起止时间

技能标签：
- 专业技能：编程语言、技术栈等
- 行业标签：所属行业和领域
```

#### 元数据字段
```
- 数据来源：固定为"脉脉招聘"
- 提取时间：数据提取的具体时间戳
- 数据准确度：系统评估的数据可信度分数
- 人才档案链接：原始页面链接（如可获取）
```

### 数据质量要求
- **准确率目标**：≥ 95%
- **完整率目标**：基础信息 ≥ 90%，详细信息 ≥ 80%
- **一致性要求**：同一字段格式保持统一
- **时效性要求**：数据提取后24小时内有效

## 🎨 用户体验要求

### 界面设计原则
- **极简主义**：界面简洁，操作直观
- **一键操作**：核心功能一键启动
- **实时反馈**：操作状态实时可见
- **错误友好**：异常情况有明确提示

### 操作流程设计
```
1. 用户访问脉脉招聘页面
2. 插件自动显示蓝色浮动控制面板
3. 用户点击"一键开始"按钮
4. 系统自动扫描、提取、翻页
5. 实时显示进度和统计信息
6. 完成后一键导出Excel文件
```

### 响应时间要求
- **界面响应**：< 200ms
- **开始处理**：< 2秒
- **单人才处理**：< 5秒
- **数据导出**：< 10秒（500人才）

## ⚡ 性能要求

### 处理能力
- **单次处理规模**：≥ 500人才
- **连续运行时间**：≥ 2小时
- **内存占用**：< 100MB
- **CPU占用**：< 10%（空闲时）

### 稳定性要求
- **崩溃率**：< 1%
- **数据丢失率**：< 0.1%
- **错误恢复率**：> 95%
- **用户操作响应率**：> 99%

## 🔒 技术约束

### 浏览器兼容性
- **主要支持**：Chrome 88+
- **测试范围**：Chrome、Edge Chromium内核
- **不支持**：Firefox、Safari、IE

### 技术限制
- **插件类型**：Chrome Extension Manifest V3
- **存储方式**：浏览器本地存储，不涉及云端
- **网络要求**：仅访问脉脉网站，无额外API调用
- **权限要求**：最小必要权限原则

### 合规要求
- **数据使用**：仅用于个人或企业内部使用
- **隐私保护**：不收集用户个人信息
- **数据安全**：本地存储，用户可自主删除
- **使用规范**：遵守脉脉网站使用条款

## 🚫 非目标（明确不做）

### 功能边界
- ❌ 不支持其他招聘网站（Boss直聘、拉勾网等）
- ❌ 不做数据分析和可视化功能
- ❌ 不做用户账号系统和云端同步
- ❌ 不做移动端适配

### 技术边界
- ❌ 不开发独立桌面应用
- ❌ 不提供API接口给第三方
- ❌ 不做实时数据同步功能
- ❌ 不集成CRM或其他外部系统
