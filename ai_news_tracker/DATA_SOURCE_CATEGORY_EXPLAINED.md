每个数据源的分类是根据其**媒体定位**和**主要内容**人工确定的：

#### 📰 **量子位** → "AI新闻"
- **依据**：量子位是专业的AI新闻媒体
- **内容特点**：报道AI行业的最新动态、新闻事件
- **分类原因**：媒体定位 = AI新闻资讯

#### 💰 **36氪** → "AI创投"
- **依据**：36氪是创投媒体，专注投融资
- **内容特点**：报道创业公司、投资、IPO
- **分类原因**：媒体定位 = 创投资讯

#### 🤖 **AI News** → "国际AI新闻"
- **依据**：国外专业AI新闻网站
- **内容特点**：全球AI行业新闻
- **分类原因**：媒体定位 + 国际来源

#### 💻 **InfoQ** → "AI技术"
- **依据**：技术深度内容平台
- **内容特点**：技术文章、架构分析
- **分类原因**：媒体定位 = 技术内容

- **内容特点**：前沿技术研究
- **分类原因**：媒体定位 = 研究/学术

## 🔄 完整流程

### 步骤1：数据源配置（人工）

```python
# 1. 人工评估数据源
# 2. 确定其媒体定位和内容特点
# 3. 分配对应的分类

"techcrunch-ai": {
    "name": "TechCrunch AI",
    "category": "AI创投",  # 人工配置：TechCrunch专注创投
}
```

### 步骤2：爬虫获取数据

```python
# backend/sources/base.py

class RSSSource(BaseSource):
    def normalize(self, raw_data: Dict[str, Any]) -> Dict[str, Any]:
        return {
            'title': raw_data.get('title'),
            'url': raw_data.get('link'),
            # ...
            'category': self.category,  # ✅ 直接使用配置的分类
        }
```

**关键点**：
- RSS本身**没有**给新闻打分类
- 程序直接使用配置文件中预设的分类
- 该数据源的所有新闻都使用同一个分类

### 步骤3：保存到数据库

```python
# backend/tasks/crawler.py

news = News(
    title=item['title'],
    category=item.get('category', 'view'),  # ✅ 来自数据源配置
    # ...
)
db.add(news)
```

## 🎯 为什么使用数据源配置分类？

### 优点

1. **快速简单** ⚡️
   - 无需API调用
   - 毫秒级完成
   - 零成本

2. **易于理解** 📚
   - 分类规则清晰
   - 符合媒体定位
   - 用户易理解

3. **稳定可靠** ✅
   - 不依赖AI服务
   - 无API限流
   - 无额外成本

4. **适合当前需求** 🎯
   - 前端只需4个聚合分类
   - 数据源配置已够用
   - 粒度粗但实用

### 缺点

1. **粒度较粗** ❌
   - 同一数据源所有新闻同一分类
   - 无法反映单条新闻的具体内容
   - 例如：36氪的AI新闻都被归为"AI创投"，即使有些是技术新闻

2. **不够精准** ❌
   - 60%准确率（基于数据源属性）
   - 可能误分类

3. **需要人工维护** 🔧
   - 新增数据源需人工配置分类
   - 数据源定位变化需更新

## 💡 未来优化方向

### 方案1：混合分类（推荐）

```python
# 第一层：数据源配置分类（当前）
category = source_config['category']  # 如 "AI新闻"

# 第二层：AI智能分类（增强）
if need_granular_classification:
    ai_result = await ai_service.classify_news(item)
    item['category'] = ai_result['category']  # 如 "model"
```

**优势**：
- 快速：数据源分类作为默认值
- 精准：AI分类提供细粒度
- 容错：AI失败时使用数据源分类

## 📝 总结

### 回答原始问题

**Q: 数据源配置分类是怎么实现的？**

**A: 人工配置！不是自动判断！**

**实现过程**：
1. **人工评估**：分析数据源的媒体定位和主要内容
2. **配置定义**：在 `SOURCES_CONFIG` 中设置 `category` 字段
3. **直接使用**：爬虫爬取时直接赋予该分类
4. **存储到数据库**：所有该数据源的新闻使用同一分类

**分类依据**：
- ✅ 数据源的**媒体定位**（新闻/技术/创投）
- ✅ 数据源的**主要内容**（报道什么）
- ✅ 数据源的**目标受众**（面向谁）
- ❌ 不是基于单条新闻的内容判断

**对比AI分类**：
- 数据源配置 = 粗分类（11个）+ 快速 + 免费
- AI智能分类 = 细分类（6个）+ 精准 + 付费
