## 🔄 迁移步骤

### 步骤 1: 创建新目录

```bash
mkdir -p docs demo archive exports/{csv,markdown,screenshots} tests/test_data extension/icons
```

### 步骤 2: 移动文档文件

```bash
# 移动所有 .md 文档到 docs/
mv PRD-AI自动筛选简历系统.md docs/
mv 开发路线图.md docs/
mv MVP-开发计划.md docs/
mv 项目进度-Day1-3总结.md docs/项目进度.md

mv 通义千问-使用指南.md docs/

mv 🚀-项目启动指南.md docs/
```

### 步骤 3: 移动 Demo 文件

```bash
mv demo_ai_resume_parser.py demo/
mv requirements-demo.txt demo/
mv DEMO-快速开始.md demo/
```

### 步骤 4: 整理插件文件

```bash
# 复制 chrome-extension-v2 到 extension
cp chrome-extension-v2/manifest.json extension/
cp chrome-extension-v2/content-full.js extension/
cp chrome-extension-v2/background.js extension/

# 复制图标文件
cp chrome-extension/icons/*.png extension/icons/

# 归档旧插件
mv chrome-extension archive/
mv chrome-extension-v2 archive/
```

### 步骤 5: 移动测试数据

```bash
mv 【专有云安全专家...】.json tests/test_data/
```


## ✅ 整理后的优点

1. **文档集中管理**：所有文档在 `docs/` 目录
2. **清晰的版本管理**：生产版插件在 `extension/`，旧版在 `archive/`
3. **数据分离**：导出文件、测试数据、日志分别存放
4. **易于维护**：目录结构清晰，易于查找和维护
5. **便于部署**：生产环境只需 `backend/` 和 `extension/`

3. **更新文档链接**：文档中的相对路径需要更新
4. **Git 提交**：整理完成后提交到 Git
