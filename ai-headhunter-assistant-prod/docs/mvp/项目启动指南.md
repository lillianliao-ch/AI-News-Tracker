## 📋 Cursor Subagent 功能介绍

### 什么是 Subagent？

**Subagent** 是 Cursor 的 AI 助手在长期项目中的"项目经理"模式：

1. **长期记忆** 📝
   - 记住你的技术选型决策
   - 记住项目架构和代码结构
   - 记住你的开发习惯

2. **模块化开发** 🧩
   - 不需要一次性完成所有代码
   - 支持分阶段、分模块逐步开发
   - 每个模块完成后自动记录进度

3. **智能续航** 🔄
   - 跨会话持续开发（今天写插件，明天写后台）
   - 自动理解上下文（不需要重复说明）
   - 基于项目历史提供针对性建议

4. **任务追踪** ✅
   - 自动管理 TODO 列表
   - 跟踪开发进度
   - 提醒遗漏的功能

## 📦 开发前准备

### 1. 安装开发环境

**必需**：
- [ ] Python 3.10+
- [ ] Node.js 18+（用于插件打包）
- [ ] PostgreSQL 15
- [ ] Redis 7
- [ ] Docker & Docker Compose

**可选**：
- [ ] VS Code / Cursor（推荐）
- [ ] Postman（测试 API）

---

### 2. 准备账号和凭证

**必需**：
- [ ] OpenAI API Key（在 https://platform.openai.com 获取）
- [ ] 飞书应用凭证（在飞书开放平台创建应用）
  - App ID
  - App Secret
  - 多维表格的 App Token 和 Table ID
- [ ] Boss 直聘账号（已登录）

**可选**：
- [ ] 腾讯云 OCR（如需处理图片简历）
- [ ] 阿里云 OSS（如需云存储简历）

---

### 3. 创建飞书多维表格

**步骤**：
1. 在飞书中创建一个新的多维表格
2. 重命名为"AI招聘-候选人库"
3. 按照 PRD 中的表结构创建字段（见 PRD 第 3.3 节）
4. 记录 App Token 和 Table ID

**字段清单**（简化版）：
```
基础字段：
- 候选人ID（文本）
- 姓名（文本）
- 匹配度（数字）
- 推荐等级（单选：推荐/一般/不推荐）
- 简历文件（附件）
- 技能标签（多选）
- 打招呼状态（单选）

详细字段见 PRD 文档第 3.3 节
```

## ❓ 常见问题

### Q1: 我没有开发经验，能用 Cursor Subagent 吗？
**A**: 可以！Subagent 会生成完整代码，你只需要：
- 提出明确的需求（"我要实现 XXX 功能"）
- 测试生成的代码
- 反馈问题（"这里不工作"）

### Q2: 每天应该花多少时间？
**A**: 建议每天 2-4 小时：
- 上午 2 小时：开发新功能
- 下午 1 小时：测试和修 bug
- 晚上 1 小时：学习相关知识

### Q3: 如果进度落后了怎么办？
**A**: 
- 优先保证核心功能（采集、解析、评估）
- 自动打招呼可以后续添加
- 某些优化功能可以放到 v1.1

### Q4: 开发过程中需要改需求怎么办？
**A**: 
- 小改动直接告诉 Subagent
- 大改动更新 PRD 文档
- Subagent 会根据新需求调整代码

### Q5: 如何知道我是否在正确的轨道上？
**A**: 每周检查点（见开发路线图）：
- Week 1: 能采集和解析简历
- Week 2: 能 AI 评估并入库
- Week 3: 能自动打招呼
- Week 4: 可以部署生产
