# 🚀 技术验证 Demo - 快速开始

## 目标

验证 AI 简历解析和评估的效果，用一个简单的 Python 脚本测试核心功能。

---

## 📦 安装依赖

### 1. 创建虚拟环境（推荐）

```bash
cd ai-headhunter-assistant

# 创建虚拟环境
python3 -m venv venv

# 激活虚拟环境
# macOS/Linux:
source venv/bin/activate

# Windows:
venv\Scripts\activate
```

### 2. 安装依赖

```bash
pip install -r requirements-demo.txt
```

**依赖说明**：
- `pdfplumber`: PDF 解析
- `openai`: OpenAI API 调用
- `Pillow`: 图像处理（PDF 依赖）

## 🎯 运行 Demo

### 基础用法

```bash
python demo_ai_resume_parser.py \
  --resume path/to/resume.pdf \
  --openai-key sk-your-api-key-here
```

### 示例

```bash
# 假设你的简历在 Downloads 目录
python demo_ai_resume_parser.py \
  --resume ~/Downloads/candidate_resume.pdf \
  --openai-key sk-proj-xxxxxxxxxxxxx
```

### 使用不同的模型

```bash
# 使用 GPT-4o（更准确，稍贵）
python demo_ai_resume_parser.py \
  --resume resume.pdf \
  --openai-key sk-xxx \
  --model gpt-4o

# 使用 GPT-4o-mini（便宜，速度快，默认）
python demo_ai_resume_parser.py \
  --resume resume.pdf \
  --openai-key sk-xxx \
  --model gpt-4o-mini
```

## ❓ 常见问题

### Q1: 提示 "ModuleNotFoundError: No module named 'pdfplumber'"

**A**: 依赖未安装，运行：
```bash
pip install -r requirements-demo.txt
```

### Q2: 提示 "OpenAI API error: Incorrect API key"

**A**: API Key 不正确，检查：
1. API Key 格式是否正确（以 `sk-` 开头）
2. API Key 是否过期
3. OpenAI 账户是否有余额

### Q3: PDF 解析结果为空

**A**: 可能原因：
1. PDF 是扫描件（需要 OCR，后续版本支持）
2. PDF 加密（需要密码）
3. PDF 格式特殊

**解决方案**：
- 尝试其他 PDF 简历
- 确保 PDF 是文字版（非扫描件）

### Q4: AI 评估结果不准确

**A**: 这是正常现象，原因：
1. Prompt 需要优化（后续会持续调优）
2. 简历内容描述不清晰
3. JD 配置与简历匹配度真的不高

**改进方法**：
- 调整 JD 配置的核心技能
- 修改评估权重
- 尝试不同的 OpenAI 模型

### Q5: 成本太高

**A**: 
- 使用 `gpt-4o-mini` 而非 `gpt-4o`（便宜 10 倍）
- 控制测试数量
- 正式版本会增加缓存机制
