# 🔴 飞书配置问题排查

## 问题现象
所有飞书 API 请求返回 `400` 错误：
```
code=91402, msg=NOTEXIST
```

## 根本原因
**`app_token` 配置错误！**

### 当前错误配置
```env
FEISHU_APP_TOKEN=XMqKwc0WCimjkIkK4fMcEtflnWd
```

从 URL 分析：
```
https://qqvh73w8uz.feishu.cn/wiki/XMqKwc0WCimjkIkK4fMcEtflnWd?table=tblQx2BAJqTAgHQA
```

- `qqvh73w8uz` - Workspace ID
- `/wiki/` - 这表明这是一个"知识库"中的表格，不是独立的多维表格
- `XMqKwc0WCimjkIkK4fMcEtflnWd` - 知识库/Wiki 的 ID
- `tblQx2BAJqTAgHQA` - 表格 ID ✅

## 飞书多维表格的两种类型

### 1. 独立多维表格 (Base)
URL 格式：`https://{workspace}.feishu.cn/base/{app_token}`
- 有独立的 `app_token`
- 可以直接通过 API 访问

### 2. 知识库中的表格 (Wiki Table)
URL 格式：`https://{workspace}.feishu.cn/wiki/{wiki_id}?table={table_id}`
- **没有独立的 `app_token`**
- `wiki_id` 不能直接用于多维表格 API
- 需要特殊的 API 路径

## 解决方案

### 方案 1: 使用独立多维表格（推荐）
1. 在飞书中创建一个新的**独立多维表格**（不是在知识库中）
2. 获取正确的 `app_token`：
   - 打开多维表格
   - 查看 URL：`https://xxx.feishu.cn/base/{app_token}`
   - 复制 `app_token`

### 方案 2: 使用 Wiki API（复杂）
如果必须使用知识库中的表格，需要使用不同的 API 端点。

## 如何获取正确的 app_token

### 步骤 1: 创建独立多维表格
1. 打开飞书 → 左侧菜单
2. 点击"+"创建 → 选择"多维表格"
3. **不要选择**"在知识库中创建"

### 步骤 2: 获取 app_token
1. 打开新创建的多维表格
2. 查看浏览器地址栏 URL
3. URL 格式应该是：
   ```
   https://{workspace}.feishu.cn/base/{app_token}
   ```
4. 复制 `/base/` 后面的那串字符，这就是正确的 `app_token`

### 步骤 3: 创建表格字段
在多维表格中创建以下字段：

| 字段名 | 字段类型 | 说明 |
|-------|---------|------|
| 候选人ID | 单行文本 | 主键 |
| 姓名 | 单行文本 | - |
| 年龄 | 数字 | - |
| 工作年限 | 数字 | - |
| 学历 | 单选 | 选项：本科/硕士/博士/大专 |
| 当前公司 | 单行文本 | - |
| 当前职位 | 单行文本 | - |
| 期望薪资 | 单行文本 | - |
| 匹配度 | 数字 | 0-100 |
| 推荐等级 | 单选 | 选项：推荐/一般/不推荐 |
| 核心优势 | 多行文本 | - |
| 潜在风险 | 多行文本 | - |
| 活跃度 | 单行文本 | - |
| 到岗状态 | 单行文本 | - |
| 简历截图 | 附件 | - |
| 数据来源 | 单选 | 选项：Boss直聘/猎聘/脉脉/LinkedIn |
| 创建时间 | 创建时间 | 自动 |

### 步骤 4: 获取 table_id
1. 在多维表格中，点击表格名称右侧的"..."
2. 选择"复制表格链接"
3. URL 中 `?table=` 后面的就是 `table_id`
