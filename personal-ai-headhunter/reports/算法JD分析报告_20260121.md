## 🔧 七、匹配系统优化记录 (2026-01-27)

### 🎯 优化目标

解决候选人与职位匹配中的"SFT污染"问题 —— 即通用技能（如SFT微调、预训练）权重过高导致不相关职位排名靠前，核心专长职位被埋没。

---

### ✅ 已完成优化

#### 1️⃣ 核心算法优化 (job_search.py)

| 功能 | 描述 | 权重 |
|:---|:---|:---:|
| **主/辅线权重体系** | 区分 `core_specialty`（核心专长）和 `tech_skills`（技术技能） | +15% / +2% |
| **标题关键词匹配** | 候选人核心专长出现在职位标题时加分 | +12% |
| **TTS/ASR冲突检测** | 防止语音合成专家匹配语音识别职位 | -15% |
| **技术方向匹配** | 宏观技术领域对齐加分 | +3%/个 |
| **标签清理函数** | `clean_tag()` 去除括号和前缀 | - |
| **同义词匹配** | `三维重建` ↔ `3D重建` 等同义词互匹配 | - |

> 📖 **详细说明**：参见 [`docs/匹配引擎权重体系说明.md`](../docs/匹配引擎权重体系说明.md)

#### 2️⃣ 标签提取优化 (extract_tags.py)

| 功能 | 描述 |
|:---|:---|
| **新增 `--force` 参数** | 支持强制重新提取所有标签 |
| **Prompt 模板更新** | 明确要求 LLM 区分 `core_specialty` 和 `tech_skills` |
| **标签体系更新** | 定义了核心专长（如语音合成、Agent开发）和技术技能（如SFT微调、预训练）的边界 |

#### 3️⃣ 全量数据更新

| 数据类型 | 数量 | 状态 |
|:---|:---:|:---:|
| **职位** | 415 | ✅ 100% 新格式 |
| **候选人** | 1,189 | ✅ 100% 新格式 |
