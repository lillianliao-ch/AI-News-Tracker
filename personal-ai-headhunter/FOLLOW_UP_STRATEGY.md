# 猎头沟通跟进策略 v1.0

> 最后更新: 2026-02-07
> 此文档记录当前的沟通策略规则，需持续优化，最终由 AI Agent 自动执行。

---

## 一、管道阶段定义

| 阶段 | 代码 | 含义 | 进入条件 |
|------|------|------|---------|
| 🆕 新发现 | `new` | 已在系统中，尚未联系 | 导入候选人时默认 |
| 📤 已打招呼 | `contacted` | 已发出第一条消息 | 插件复制/填入消息时自动设置 |
| 🔄 跟进中 | `following_up` | 未回复，正在2/3次跟进 | 手动点击"再跟进" |
| 💬 已回复 | `replied` | 对方回复了消息 | 手动点击"已回复"，或添加 inbound 沟通记录时自动触发 |
| 💚 已加微信 | `wechat_connected` | 拿到微信号，转微信沟通 | 手动点击"加微信" |
| 🎯 面试中 | `in_pipeline` | 已推荐给客户，进入面试流程 | 手动标记 |
| ⏸️ 关闭 | `closed` | 暂时不跟进 | 手动点击"关闭" |

---

## 二、自动跟进规则

| 触发事件 | 自动设置的阶段 | 跟进天数 | 说明 |
|---------|-------------|---------|------|
| 首次打招呼（插件发消息） | `contacted` | **3天** | 3天后提醒查看是否回复 |
| 点击"再跟进" | `following_up` | **5天** | 给更多时间，避免太频繁 |
| 点击"已回复" | `replied` | **1天** | 趁热打铁，尽快要微信 |
| 点击"加微信" | `wechat_connected` | **2天** | 加上微信后尽快推JD |
| 进入面试流程 | `in_pipeline` | **3天** | 跟进面试进度 |
| 点击"关闭" | `closed` | 无 | 暂不跟进 |

### 待优化点
- [ ] 跟进次数上限：跟进3次仍无回复 → 自动关闭
- [ ] 换渠道策略：脉脉无回复 → 自动建议用 LinkedIn 或邮件
- [ ] 节假日跳过：跟进日期避开周末和法定假日
- [ ] 候选人活跃度：根据"近1月活跃"等标签调整跟进频率

## 三、多渠道触达策略

### 渠道优先级
1. **脉脉** — 国内AI/互联网候选人主阵地，初次触达首选
2. **LinkedIn** — 海外/外企候选人，或脉脉无回复的补充渠道
3. **邮件** — 正式沟通、发送JD详情
4. **微信** — 深度沟通、推JD、约电话、持续维护关系
5. **电话** — 正式沟通、面试辅导

### 渠道切换规则（待实现）
- 脉脉打招呼 3天无回复 → 尝试 LinkedIn
- LinkedIn 5天无回复 → 尝试邮件（如有）
- 所有渠道无回复 → 关闭，30天后可重新激活

## 四、AI消息生成规则

### 称呼规则（代码判断，非LLM）
- 2-4字中文名 → 姓+先生（如"王文轩"→"王先生"）
- 2字中文名 → 全名+您好（如"王伟"→"王伟您好"）
- 昵称/英文名/网名 → "您好"

### 内容规则
1. 自我介绍：Lillian，专注AI/互联网/高科技领域的猎头
2. 提到对方具体亮点（公司+年限+方向），不泛泛而谈
3. 准确判断核心方向：以全职经历为主，兼职/众包不当主要亮点
4. JD关联：不直接推JD，说"我们有一些机会跟您比较匹配"
5. 行动呼吁："希望能与您进一步交流"
6. 末尾附联系方式：电话/微信 13585841775

### 禁止规则
- 🚫 "虽然...但是/不一致/不匹配" 等负面句式
- 🚫 "巨大价值/非常匹配/非常丰富/令人印象深刻" 等空洞吹捧
- 🚫 "祝好/Best regards" 等签名
- 🚫 分点列举，必须写成自然的一段话
- 🚫 超过250字

## 五、沟通记录格式

每条沟通记录包含：
```json
{
  "time": "2026-02-07 19:30",
  "channel": "maimai",        // maimai | linkedin | email | wechat | phone
  "action": "greeting",       // greeting | follow_up | manual_log | stage_change
  "content": "消息内容",
  "direction": "outbound"     // outbound | inbound | system
}
```

### 记录来源
| 来源 | 记录方式 | channel |
|------|---------|---------|
| 脉脉插件复制/填入消息 | 自动 | `maimai_direct` |
| 候选人详情页手动添加 | 手动选择渠道和方向 | 用户选择 |
| 管道阶段变更 | 自动 | `system` |

## 六、未来自动化路线图

### Phase 1 ✅ 当前版本
- 打招呼后自动记录 + 自动设跟进日期
- 沟通跟进看板（策略跟进 + 手动预约）
- 手动阶段流转按钮

### Phase 2 🔜 半自动化
- 跟进消息AI生成（根据上次消息内容和间隔天数）
- 多渠道策略建议（推荐下一个触达渠道）
- 批量跟进操作
- 统一跟进系统（合并策略跟进和手动预约）

### Phase 3 🎯 全自动化 Agent
- Agent 自动判断应该跟进谁
- Agent 自动生成跟进消息（根据阶段、渠道、历史）
- Agent 自动切换渠道
- Agent 自动判断是否关闭（多次无回复）
- 每日运营报告自动生成
