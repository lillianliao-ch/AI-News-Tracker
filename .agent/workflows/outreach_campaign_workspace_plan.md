# 触达作战中心 (Campaign Workspace) 实施规划

基于对现有系统 UI（人才大列表、档案侧滑窗、每日工作台、沟通跟进大盘）的图文分析，本次重构的核心哲学是：**严格分离“找人 (Sourcing)”与“联系 (Outreach)”，在保护现有沉浸式浏览体验的前提下，利用已有组件构建工业级发件流水线。**

## 1. 架构定位：与现有页面的融合

从你提供的截图来看，系统目前拥有：
*   `【人才列表】`：用于找人（图1、图2）—— **定调：保持原样，纯净的搜索与浏览**。
*   `【每日工作台】`：用于宏观数据盯盘（图4）—— **定调：保持原样**。
*   `【沟通跟进】`：用于管理发信后的状态流转（图3，包含“新发现、已打招呼、已回复”等 Tab，以及“策略跟进”横向列表）—— **定调：本次升级的核心改造区**。

本次规划将直接把**发件与唤醒**功能，深度融合进 `【沟通跟进】` 这个模块（或者作为其平行的新顶级菜单），彻底打通这最后一百米。

---

## 2. 核心模块设计说明

### 模块 A：LinkedIn 精准狙击区 (Split-Pane Context View)
*   **入口位置**：在「沟通跟进」大盘中，新增一个专属的子菜单或独立视图 `[🎯 LinkedIn 精准触达]`。
*   **适用场景**：必须看简历才能决定发不发、怎么发的 S/A 级高端局。
*   **UI 创新 (左右分栏布局)**：
    *   **左侧 (30% 宽度) - 任务队列**：列出当前筛选条件下（如 `tier:S` + `未触达` + `拥有LinkedIn`）的候选人列表（仅显示姓名、公司、Title）。点击可在右侧切换。
    *   **右侧 (70% 宽度) - 档案与发件台**：
        1.  **上半部分**：直接复用你【图2】中那套非常成熟的组件（包含学历、经历、Repo 等所有研判信息）。
        2.  **顶部特权区**：悬浮一个高亮的文本框，里面已经填好了后台跑好的 **LLM 150字专属破冰信 (`greeting_drafts`)**。
        3.  **行动按钮**：`[一键复制并打开 LinkedIn 主页]` 👉 你去网页发完后，回来点击 `[✅ 标记已打招呼 (LinkedIn)]`。
*   **流转结果**：此人瞬间从左侧队列消失，底层数据进入 `已打招呼` 状态，出现在你【图3】的横向跟进列表里。

### 模块 B：Gmail 批量轰炸机 (Batch Email Table)
*   **入口位置**：新增子菜单 `[📧 Email 批量战役]`。
*   **适用场景**：只看 Tier 和标签，不需要细读履历，靠量取胜的一般/中高端岗位。
*   **UI 设计 (极简数据表)**：
    *   **显示字段**：`[复选框] | 姓名 | 当前公司 | 匹配邮箱 | Tier | 邮件标题预览 | 本次战役状态`。
    *   **批量操作条**：放置在表格顶部或底部悬浮。包含最核心的大按钮 `[🚀 批量发送所选 (Send X Emails)]`。
    *   **直观修改**：点击表格中的“邮件标题预览”单元格，支持行内弹窗快速小修小补 LLM 生成的草稿。
*   **流转结果**：一键点击后，后端静默调用 Gmail API。成功发送的人同样会流入 `已打招呼` 状态，并在【图3】的看板中增加对应数字。

### 模块 C：沉默唤醒雷达 (Nurture & Follow-up Radar)
*   **定位**：对你【图3】中现有的“策略跟进”列表进行逻辑和交互升级。
*   **痛点解法 (Stop Rule 机制)**：
    *   现在的红标警告（如“逾期13天”）非常好。需要增加**动作建议**和**终结机制**。
    *   当某人逾期 `>3天` 未回，系统在红标旁提供快捷按钮：`[发第二封信 (Bump)]`。
    *   当此人的 `已联系 X 次` 达到 **3次** 时（Stop Rule 触发），卡片变灰，红标变为 `[💡 建议移入冰冻池]`。一键点击后，他在接下来的半年内不再出现在这个页面，还跟进面板真正的清爽。

## 3. 技术落地路线图

此架构不需要对后端底层进行大改（因为 `pipeline_stage`, `greeting_drafts`, `last_outreach_date`，`outreach_count` 等底层字段已于前序迭代中准备完毕）。

1.  **里程碑 1 (前端路由与骨架)**：在 React 中创建新的 `Campaigns` 路由群组，搭建 Split-Pane 和 Table 两个基础视图骨架。
2.  **里程碑 2 (LinkedIn 分栏实装)**：将现有的 Candidate Profile 组件注入右侧栏，并渲染 `greeting_drafts` 内容和一键复制/标记逻辑。
3.  **里程碑 3 (Gmail 批量实装)**：开发带批量勾选状态管理的 Table 组件，并对接后端的 `/api/batch_send_email` 接口。
4.  **里程碑 4 (看板与跟进重构)**：基于现有的【图3】沟通大盘，增加 Stop Rule 前端判定逻辑和冰冻移出操作。
