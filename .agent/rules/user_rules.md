# User Rules & Preferences

## Iterative Development Protocol
1.  **Stop Point**: After proposing a **Solution** or **Implementation Plan**, you MUST stop and wait for explicit verbal confirmation (e.g., "Go ahead", "Start", "Proceed with plan") from the user.
2.  **No Assumptions**: Approval of a document (e.g., "LGTM" on a plan) does NOT automatically authorize the start of coding. Always ask: "Shall I proceed with implementation?" and wait for the answer.
3.  **Step-by-Step**: When significant architectural changes are proposed, confirm the approach first before writing any code.
